# Contributing to Awesome Engineering Leadership

Thank you for your interest in contributing! This list is community-driven, and we welcome additions from engineering leaders at all levels.

## How to Contribute

### Adding a Resource

1. **Fork** this repository
2. **Edit** `README.md` to add your resource in the appropriate section
3. **Submit** a pull request with a clear description of what you're adding and why

### Resource Format

Each resource should follow this format:

```markdown
- [Title](URL) by *Author* — Brief description (under 200 characters)
```

For books:
```markdown
- [Book Title](URL) by *Author Name* — One-line description of the book's focus and value.
```

For articles:
```markdown
- [Article Title](URL) by *Author Name* — One-line description of the article's key insight.
```

### Quality Guidelines

Before submitting, please ensure your resource meets these criteria:

- **Relevant**: Directly related to engineering leadership, management, or technical leadership
- **High quality**: Well-written, insightful, and actionable
- **Accessible**: Available online (not behind a paywall, or clearly marked if it is)
- **Not a duplicate**: Check that the resource isn't already in the list
- **Correct section**: Place the resource in the appropriate type section (Books, Articles, Videos, etc.)
- **Alphabetical order**: Resources within each section are sorted alphabetically by title

### Adding Agent Skills

Agent Skills are curated from [SkillsMP](https://skillsmp.com). To suggest a new skill:

1. Ensure the skill is published on SkillsMP
2. Add it to the appropriate leader-type bucket in the Agent Skills section
3. Follow the format: `- [skill-name](skillsmp-url) by *author* ⭐ stars — Description`

### Reporting Issues

- **Broken links**: Open an issue with the broken URL and suggested replacement
- **Incorrect information**: Open an issue or PR with the correction
- **Category suggestions**: Open an issue to discuss new categories or reorganization
